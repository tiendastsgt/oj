## 1. RESUMEN EJECUTIVO

### 1.1 ¿Qué es este módulo?

Sistema de **OCR + Extracción Inteligente** para documentos (facturas, recibos, contratos, certificaciones, expedientes) que combina:

- **OCR local** (Tesseract) para extraer texto de PDF/imágenes
- **LLM local** (Ollama/Qwen) para clasificación y estructuración JSON
- **LLM remoto** (DeepSeek API) como alternativa de alta calidad

### 1.2 Capacidades Actuales

| Capacidad | Estado | Descripción |

| OCR de imágenes | ✅ Funciona | JPG, PNG → texto |

| Idiomas | ✅ Configurado | Español + Inglés |
| Clasificación automática | ✅ Funciona | Detecta tipo de documento |
| Extracción JSON (local) | ✅ Funciona | Ollama + qwen2.5-coder:7b |
| Extracción JSON (API) | ✅ Funciona | DeepSeek API |
| Flujo automatizado | ✅ Funciona | Script process-document.sh |

## 2. SERVICIOS DESPLEGADOS

### 2.1 ocr-tesseract (Puerto 8101)

**Función:** Extrae texto de documentos PDF e imágenes usando Tesseract OCR.

**Contenedor:** `ocr-tesseract`

**Imagen:** `sts-ocr-tesseract`

**Configuración:**

| Variable | Valor | Descripción |
|----------|-------|-------------|
| Puerto | 127.0.0.1:8101 | Solo localhost |
| Idiomas | eng+spa | Inglés y Español |
| Motor | Tesseract | OCR de código abierto |

**Endpoints:**

| Método | Ruta | Descripción |
|--------|------|-------------|
| GET | `/health` | Health check |
| POST | `/ocr` | Procesar documento |

**Request `/ocr`:**
```bash
curl -X POST http://localhost:8101/ocr \
  -F "file=@documento.pdf" \
  -F "lang=eng+spa" \
  -F "dpi=200"
```

**Response:**
```json
{
  "filename": "documento.pdf",
  "pages": [
    {
      "page": 1,
      "lines": [
        {"text": "FACTURA #12345", "confidence": 0.95},
        {"text": "Total: $1,250.00", "confidence": 0.92}
      ]
    }
  ],
  "text": "FACTURA #12345\nTotal: $1,250.00\n..."
}
```

**Ubicación del código:** `~/ocr-tesseract/`

### 2.2 llm-ollama-proxy (Puerto 8202)

**Función:** Proxy para clasificación y extracción estructurada usando Ollama local (GRATIS).

**Contenedor:** `llm-ollama-proxy`

**Imagen:** `sts-llm-ollama-proxy`

**Versión:** 1.3

**Configuración:**

| Variable | Valor | Descripción |
|----------|-------|-------------|
| Puerto | 127.0.0.1:8202 | Solo localhost |
| OLLAMA_BASE_URL | http://conta-ollama:11434 | Ollama interno |
| OLLAMA_MODEL | qwen2.5-coder:7b | Modelo por defecto |
| OLLAMA_TIMEOUT | 300 | Timeout en segundos |

**Endpoints:**

| Método | Ruta | Descripción |
|--------|------|-------------|
| GET | `/health` | Health check |
| GET | `/templates` | Lista tipos de documentos |
| POST | `/extract` | Extracción con instrucción custom |
| POST | `/analyze` | Clasificación + extracción automática |

**Request `/analyze`:**
```bash
curl -X POST http://localhost:8202/analyze \
  -H "Content-Type: application/json" \
  -d '{
    "text": "CONTRATO de servicios entre Empresa A y B...",
    "document_type": "contract"
  }'
```

**Response:**
```json
{
  "document_type": "contract",
  "confidence": 0.85,
  "data": {
    "title": "SAMPLE AGREEMENT",
    "effective_date": "XX day of October, 200X",
    "parties": [
      {"role": "Committee", "name": null},
      {"role": "Contractor", "name": null}
    ],
    "sections": ["Independent Contractor", "Duties", "Expenses"]
  },
  "model": "qwen2.5-coder:7b"
}
```

**Tipos de documento soportados:**

| Tipo | Campos extraídos |
|------|------------------|
| `invoice` | invoice_number, date, vendor_name, client_name, items, subtotal, tax, total, currency |
| `purchase_order` | po_number, date, buyer_name, vendor_name, items, total, terms |
| `contract` | title, effective_date, parties, sections, termination_terms |
| `certificate` | certificate_type, issue_date, expiry_date, issued_to, issued_by, certificate_number |
| `generic` | document_type, title, date, key_information |

**Ubicación del código:** `~/llm-ollama-proxy/`

## 3. FLUJO AUTOMATIZADO

### 3.1 Script process-document.sh

**Ubicación:** `~/process-document.sh`

**Función:** Orquesta OCR + LLM en un solo comando, sin intervención humana.

**Uso:**
```bash
# Auto-detectar tipo de documento
~/process-document.sh /ruta/documento.pdf

# Forzar tipo de documento
~/process-document.sh /ruta/factura.pdf invoice
```

**Tipos soportados:** `invoice`, `purchase_order`, `contract`, `certificate`, `generic`

**Ejemplo:**
```bash
~/process-document.sh /tmp/contrato.jpg | jq
```

**Salida:**
```json
{
  "document_type": "contract",
  "confidence": 0.85,
  "data": {
    "title": "SAMPLE AGREEMENT",
    "effective_date": "XX day of October, 200X",
    "parties": [
      {"role": "Committee", "name": null},
      {"role": "Contractor", "name": null}
    ],
    "sections": ["Independent Contractor", "Duties", "Expenses"]
  },
  "model": "qwen2.5-coder:7b"
}
```


## 8. TROUBLESHOOTING

### 8.1 Error "Ollama timeout"

**Causa:** El modelo tarda más del timeout configurado.

**Soluciones:**
1. Pre-calentar el modelo (ver sección 5.4)
2. Verificar memoria disponible: `free -h`
3. Aumentar timeout al recrear contenedor:
```bash
docker run -d --name llm-ollama-proxy \
  ... \
  -e OLLAMA_TIMEOUT=600 \
  ...
```

### 8.2 Error "JSON decode error"

**Causa:** Caracteres especiales en el texto.

**Solución:** El script `process-document.sh` ya usa `jq --rawfile` para escapar correctamente.

### 8.3 OCR devuelve texto vacío

**Causas:**
- Imagen muy pequeña o borrosa
- PDF escaneado sin texto
- Idioma incorrecto

**Solución:**
```bash
curl -X POST http://localhost:8101/ocr \
  -F "file=@documento.pdf" \
  -F "lang=spa" \
  -F "dpi=300"
```

### 8.4 Memoria insuficiente

Con 7.8 GB RAM, el modelo qwen2.5-coder:7b consume ~5 GB.

```bash
# Ver uso de memoria
free -h
docker stats --no-stream
```
