# Capitulo 1 — Introduccion al sistema

## Objetivo de este capitulo

Al terminar de leer este capitulo usted sabra:
- Que es el SGED y para que sirve.
- Desde que dispositivos y navegadores puede usarlo.
- Que puede hacer cada tipo de usuario dentro del sistema.

---

## 1.1 Que es el SGED

El **SGED** (Sistema de Gestion de Expedientes Digitales) es una plataforma web institucional que permite organizar, consultar y gestionar expedientes judiciales y sus documentos de respaldo en un unico lugar seguro y accesible.

Con el SGED usted puede:

- Crear y mantener actualizado el registro de expedientes judiciales.
- Adjuntar documentos de distintos formatos (PDFs, imagenes, audios, videos, hojas de calculo, documentos de texto) directamente desde su computadora.
- Visualizar cualquier documento en pantalla sin necesidad de descargarlo.
- Buscar expedientes por multiples criterios: numero, tipo de proceso, estado, juzgado, partes involucradas y fechas.
- Consultar indicadores de gestion desde el panel principal.

El sistema opera completamente en el navegador. No requiere instalar ningun programa adicional en su computadora.

---

## 1.2 Requisitos del navegador

Para usar el SGED correctamente, su computadora debe contar con uno de los siguientes navegadores actualizados:

| Navegador | Version minima recomendada |
|-----------|---------------------------|
| Google Chrome | Version 110 o superior |
| Mozilla Firefox | Version 110 o superior |
| Microsoft Edge | Version 110 o superior |
| Safari (macOS) | Version 16 o superior |

> [!WARNING]
> **No use Internet Explorer.** Este navegador es obsoleto y no es compatible con el SGED. Si su computadora solo tiene Internet Explorer, solicite al area de informatica que instale un navegador moderno.

> [!NOTE]
> Se recomienda usar el sistema con una resolucion de pantalla de al menos 1280 x 720 pixeles para una experiencia optima. En pantallas mas pequenas el sistema es funcional pero el espacio de trabajo se reduce.

## 1.3 Tabla de permisos por rol

El SGED tiene cuatro roles de usuario. Cada rol determina exactamente que acciones puede realizar dentro del sistema. La siguiente tabla resume los permisos generales:

| Accion | ADMIN | SECRETARIO | AUXILIAR | CONSULTA |
|--------|:-----:|:----------:|:--------:|:--------:|
| Iniciar sesion | Si | Si | Si | Si |
| Ver panel principal (Dashboard) | Si | Si | Si | Si |
| Ver listado de expedientes | Si | Si | Si | Si |
| Crear expedientes | Si | Si | Si | No |
| Editar datos de expedientes | Si | Si | No | No |
| Eliminar expedientes | Si | No | No | No |
| Ver documentos de un expediente | Si | Si | Si | Si |
| Subir documentos | Si | Si | Si | No |
| Descargar documentos | Si | Si | Si | Si |
| Eliminar documentos | Si | Si | No | No |
| Usar busqueda rapida | Si | Si | Si | Si |
| Usar busqueda avanzada | Si | Si | Si | Si |
| Gestionar usuarios del sistema | Si | No | No | No |
| Administrar catalogos (juzgados, tipos de proceso, estados) | Si | No | No | No |

> [!TIP]
> Si un boton o menu no aparece en su pantalla, probablemente su rol no tiene permiso para esa accion. Esto es normal y forma parte del control de seguridad del sistema. Consulte con su supervisor si cree que necesita un permiso adicional.

## 1.4 Glosario de terminos basicos

Para entender este manual no necesita conocimientos tecnicos. A continuacion se definen los terminos que se usaran a lo largo de los capitulos:

| Termino | Significado en el SGED |
|---------|----------------------|
| **Expediente** | Carpeta digital que agrupa todos los documentos e informacion de un caso judicial |
| **Documento** | Archivo adjunto dentro de un expediente (PDF, imagen, audio, video, etc.) |
| **Visor** | Herramienta integrada en el sistema que permite ver el contenido de un documento en pantalla |
| **Dashboard** | Panel principal con indicadores de gestion que se muestra al iniciar sesion |
| **Rol** | Conjunto de permisos asignado a un usuario que define lo que puede hacer en el sistema |
| **Estado** | Situacion actual de un expediente: Activo, Archivado o Suspendido |
| **Sesion** | Periodo activo de trabajo en el sistema, desde que ingresa hasta que cierra sesion o esta expira |
